What is it?

Oli aims to be a confortable minimal language for multiple purposes which provides a mix of features from common markup languages and some basic features from programming languages

It has a beutiful, elegant and clean syntax with unobstructive grammar, that allows to use it for multiple purposes, like using it to create your own DSL. It was designed to be mainly consumed by humans and keeping sintax extensibility in mind

Features

Primitives Types

Primitive types with powerful type inference: boolean, number and string

Data Structures

Lists and data hash maps has first class support

Linked data

Built-in support for data link references inside the document

DRY

Copy or inherits data structures and values by reference (it also support multiple inheritance)

Templating

Powerful templating features including string interpolation

Higienic syntax

Clean and unobstructive syntax grammar-free with only two reserved keywords

Oli is useful for

>> Create syntax clean and readable configuration files

>> Use it as your own DSL for specific purposes

>> Store human writable/readable data, like manifest files

>> Data interchange format for non-complex schemas structures

>> Featured markup-like language for templating

Design Goals

>> Human focused, readability matters

>> Keep simple as possible, but provide built-in rich features

>> Permissive syntax but realiable, avoid ambiguity

>> Unobstructive minimal syntax grammar-free, ideally for DSL

>> Elegant mix of common programming languages features

Oli in the real world

>> Buildspec - Declarative, featured, CI-agnostic build configuration file

>> Olite - Markup-like templating engine which runs in JavaScript environments
