What is it?

Oli aims to be a confortable minimal language for multiple purposes which provides a mix of features from common markup languages and some basic features from programming languages

It has a beutiful elegant and clean syntax with unobstructive grammar, that allows you to use it for multiple purposes like creating your own DSL adapting the syntax for the specific use case context. It was designed to be mainly consumed by humans

Features

Primitives Types

Primitive types with powerful type inference: boolean, number and string

Data Structures

Lists and data hash maps has first class support

Linked data

Built-in support for data link references inside the document

DRY

Copy or inherits data structures and values by reference (it also support multiple inheritance)

Templating

Powerful templating features including string interpolation

Higienic syntax

Clean and unobstructive syntax grammar-free (symbol-based, it just have only two reserved keywords)

Upcoming features

Indent-based blocks

A clean and short way to define blocks. "end" terminator token will be optional

Block scope references

References scope will only be applied inside the block it was defined and child blocks

Interpolated code

Built-in support for interpolated code expressions allowing control-flow structures and iteration

Functions

Helper functions for data transformation and formatting

Oli is useful for

>> Create syntax clean and readable configuration files

>> Use it as your own DSL for specific purposes

>> Store human writable/readable data, like manifest files

>> Data interchange format for non-complex schemas structures

>> Featured markup-like language for templating

Design Goals

>> Human focused, readability matters

>> Keep it simple but provide built-in rich features

>> Realiable syntax, avoid ambiguity

>> Unobstructive grammar-free syntax

>> Elegant mix of basic programming languages features
