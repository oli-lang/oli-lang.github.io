What is it?

Oli aims to be a confortable minimal language for multiple purposes which provides a mix of features from common markup languages and some basic features from programming languages

It has a beautiful, elegant and clean syntax with unobstructive grammar, that allows you to use it for multiple purposes like creating your own DSL adapting the syntax for the specific use case context. It was designed to be mainly consumed by humans

Features

Primitive Types

Primitive types with powerful type inference: boolean, number and string

Data Structures

List and data maps had first class support

Linked Data

Built-in support for data link references inside the document

DRY

It provides built-in support for copy and inheritance of data structures

Templating

Powerful templating features including string interpolation

Higienic Syntax

Clean and unobstructive syntax grammar-free symbol-based

Upcoming Features

Indent-based Blocks

A clean and short way to define blocks. "end" terminator token will be optional

Block Scope

Each new block creates a new referenceable scope which inherits from parents blocks

Interpolated Code

Built-in support for interpolated code expressions allowing control-flow structures and iteration

Functions and Math operations

Helper functions for data transformation, formatting and math basic operations

Oli is useful for

>> Creating clean and readable configuration files

>> Using it as your own DSL for specific purposes

>> Storing human writable/readable data, like manifest files

>> As data interchange format for non-complex schemas structures

>> Featured markup-like language for templating

>> Partial replacement for JSON, XML, INI...

Design Goals

>> Human focused, readability matters

>> Keep it simple but provide built-in rich features

>> Realiable syntax. Avoid ambiguity

>> Unobstructive grammar-free syntax

>> Elegant mix of basic programming languages features
