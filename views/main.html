What it is?

Oli aims to be a confortable minimal language for general purposes which provides a mix of features from common markup languages and some basic features from programming languages

It has a beutiful, elegant and clean syntax with unobstructive grammar, that allows to use it for multiple purposes, like creating your own DSL. It was designed to be mainly consumed by humans

Features

Primitives Types

Support for trivial primitive types: boolean, number and string

Data Structures

Nested lists and data hashmaps has first class support

References

Built-in support for data references inside the document

DRY

Copy or extend data structures and values by reference

Templating

Powerful templating features including string interpolation

Operators

Logical, comparison and basic math operators

Oli is useful for

>> Create sintax clean and readable configuration files

>> Create you own DSL for specific purposes

>> Store human writable/readable data, like manifest files

>> Data interchange format for non-complex schemas structures

>> Powerful markup-like templating language

Design Goals

>> Human focused, readability matters

>> Keep simple as possible, but provide built-in rich features

>> Permissive sintax but realiable, avoid ambiguity

>> Unobstructive minimal sintax grammar-free, ideally for DSL

>> Elegant mix of common programming languages features

Contributing

Oli is a community-driven project. The language features and specification are under active debate and enhancement process. It's open for everyone to discuss
